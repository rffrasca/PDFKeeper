Setup Single-User Database

SQLite is the database engine used by PDFKeeper for single-user document storage. It provides an easy to setup database option that does not require a database management system.

The single-user database is contained in a file named PDFKeeper.sqlite that is located in %APPDATA%\Robert F. Frasca\PDFKeeper. This database will need to be backed up regularly in the event the database needs to be restored due to corruption or accidental data loss.

You are fully responsible for this database and any additional databases you create!

The default location for the database may not be ideal for some system configurations and can be moved to an alternate location.

Create a New Single‑User Database

You can create additional single‑user databases—for example, to separate personal and work documents or to start a fresh archive.

To create a new database:

  1. From the PDFKeeper main form, select File → Database → New Database, or press Ctrl + Shift, then N.

  2. Select the folder where the new database should be created.

  3. Accept the default or enter a different name for the database file.

  4. Select Save or press Enter to create the new database.

  5. PDFKeeper will automatically switch to the newly created database.

Open an Existing Single‑User Database

If you have an existing PDFKeeper (SQLite) database—either previously created or restored from backup—you can open it at any time.

To open an existing database:

  1. From the PDFKeeper main form, select File → Database → Open Database, or press Ctrl + Shift, then O.

  2. Browse to the folder containing the database file.

  3. Select the PDFKeeper.sqlite file (or another valid PDFKeeper (SQLite) database file).

  4. Select Open or press Enter to reference the selected database.

  5. PDFKeeper will automatically switch to the selected database.

Move the Database

To move the current database to a different location:

  1. From the PDFKeeper main form, select Tools → Move Database, or press Alt + T, then M.

  2. Select the folder where the database should be moved.

  3. Select OK or press Enter to perform the move.
