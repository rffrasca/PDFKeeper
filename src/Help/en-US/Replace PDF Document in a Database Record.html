Replace PDF Document in a Database Record

This topic explains how to replace an existing PDF file stored in a database record with a new version.

Steps to Replace a PDF Document:

  1. From the PDFKeeper main form, select the document record that contains the PDF you want to replace.

  2. Open the Edit menu, then select Replace PDF, or press Ctrl + E on the keyboard.

  3. Browse to the PDF you want to upload, select it, and then click Open or press Enter.

  4. If the selected PDF is protected with an OWNER password, you will be prompted to enter the correct password. If you do not know the password but have the legal right to remove it, you can find guidance online.

  5. The Replace PDF form will open.

  6. To view the selected PDF, select View or press Alt + V to open the PDF in the bundled viewer.

  7. Type a Title for the PDF. To automatically set the title to the file name, select Set Title to PDF File Name.

  8. Use the Author combo box to type a new author or select one from the database.

  9. Use the Subject combo box to type a new subject or select one from the database.

  10. Add Keywords if needed.

  11. Use the Category combo box to type a new category or select one from the database if required.

  12. Use the Tax Year combo box to choose the applicable tax year if required.

  13. Check or uncheck Flag Document. Example: Flagging a bill makes it easy to list documents that require payment.

  14. Check or uncheck OCR PDF pages containing both text and image data. When checked, pages with embedded images are processed by OCR to extract text that would otherwise be ignored.

  15. Check or uncheck Delete selected PDF when added. When checked, the original PDF is moved to the Windows Recycle Bin after being added.

  16. Select OK or press Enter to save the modified PDF with the specified Information Properties and external metadata XML to %APPDATA%\Robert F. Frasca\PDFKeeper\UploadStaging. Note: If the selected PDF contained an OWNER password, it will not be written to the modified PDF. To cancel, select Cancel or press Esc.

Additional Information:

  • In a multi-user database environment, the controls referenced in step 2 will be disabled if your database user account has not been granted update access to the docs table.
