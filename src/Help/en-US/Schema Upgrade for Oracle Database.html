Schema Upgrade for Oracle Database

This topic provides the steps that need to be performed by the individual responsible for the administration of the Oracle Database when upgrading from PDFKeeper 6.1.1 or earlier. This schema upgrade adds support for new features introduced in PDFKeeper 5.0.0 and 7.0.0.

Before proceeding, it’s highly recommended that the Oracle Database be backed using RMAN, then backup the Fast Recovery Area to an alternate location. For information on backing up the database using RMAN, please consult the online documentation for the database version being used.

  1. If the Oracle Database used to store PDFKeeper data was installed on a different computer, you will need to download Oracle Instant Client Basic or Basic Lite and SQL*Plus packages to this computer, extract the ZIP files into an empty folder, and then add that folder to the PATH; however, you can skip this if you can open a command prompt, and then type sqlplus to confirm the command is available. As an alternative, you can install this version of PDFKeeper on the same computer as the Oracle Database used to store PDFKeeper data, and then perform the schema upgrade on that computer.

  2. Perform the schema upgrade by selecting option 3 from the menu.

  3. See the Updating PDF Text Columns in the Database help topic for instructions on how to update the newly added database table columns.
