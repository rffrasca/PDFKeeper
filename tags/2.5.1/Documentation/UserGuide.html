1. Introduction

What is PDFKeeper?
It is a free, open source PDF document storage system that provides storage, indexing, and retrieval using the free Oracle Database Express Edition (Oracle Database XE).
PDFKeeper is perfect for anyone who wants to implement a low cost PDF document storage system in a small or home office. It can be implemented for use on a single user system or in a workgroup.
About this user guide
This guide provides the steps that need to be performed in order to implement and use PDFKeeper. It is intended to be used by those involved in implementing, administering, and using PDFKeeper. Before proceeding, make sure that all software required by PDFKeeper has been installed.

2. Database Setup

Description
The Database Setup script creates the following:
  • pdfkeeper schema account in the users tablespace.
  • docs sequence used to generate document ID numbers, in the pdfkeeper schema.
  • docs table used to store the documents, in the pdfkeeper schema.
  • Oracle Text, lexer and datastore preferences; and attributes, in the ctxsys schema.
  • docs table full-text search index, in the pdfkeeper schema.
Table structure
Each document is stored in the database as follows:
ID - Assigned to the document record by the database.

Title - Extracted from the PDF summary (InfoDictionary). This column is searchable.

Author - Extracted from the PDF summary (InfoDictionary). This column is searchable.
Subject - Extracted from the PDF summary (InfoDictionary). This column is searchable.
Keywords - Extracted from the PDF summary (InfoDictionary). This column is searchable.
Added - Date and time the document was added to the database. This column is searchable.
Notes - User editable notes. This column is searchable.
PDF - The contents of the PDF document. This column is searchable.
Action steps
  1. Select Start | All Programs | PDFKeeper | Database Setup.
  2. When prompted, enter the database connect string in the format: username@host:port
    • username is a database account that must be a member of SYSDBA. Note, SYSTEM is a member of SYSDBA.
    • host is the host name or IP address of the computer that is running Oracle Database XE Server. Specify localhost or 127.0.0.1 if logged onto the same system as Oracle Database XE Server.
    • port (optional) is the TCP port number on which the Oracle Net listener is listening. If not specified, the default port number 1521 is assumed.
  3. Secure the pdfkeeper database user account by changing the password, the default password is pdfkeeper.
    1. On the system that has Oracle Server installed, log into Windows as the user that installed Oracle Server.
    2. Select Start | All Programs | Oracle Database (10g or 11g) Express Edition | Run SQL Command Line.
    3. When prompted, enter the following: connect / as sysdba
    4. Next, enter the following: alter user pdfkeeper identified by new_password;

3. Database User Account Management

Recommendation
It is recommended that each user be assigned a personal database account, with only the required privileges. The pdfkeeper database account should only be used for pdfkeeper schema administration, not for general use. For more information on Managing Users, please refer to the Oracle Database 11g Express Edition 2 Day DBA guide or Oracle Database 10g Express Edition 2 Day DBA guide.
Granting access to the pdfkeeper schema
Perform the following steps for any user that will need access to the pdfkeeper schema:
  1. If the user does not have a database user account, create one for the user:
    1. On the system that has Oracle Server installed, log into Windows as the user that installed Oracle Server.
    2. Select Start | All Programs | Oracle Database (10g or 11g) Express Edition | Run SQL Command Line.
    3. When prompted, enter the following: connect / as sysdba
    4. Next, enter the following: create user user_name default tablespace users identified by password;
    5. Next, enter the following: grant connect, resource to user_name;
    6. Close SQL Command Line, enter: quit
  2. Connect to Oracle Server as pdfkeeper
    1. Select Start | All Programs | Oracle Database (10g or 11g) Express Edition | Run SQL Command Line.
    2. When prompted, enter the following: connect pdfkeeper
    3. When prompted, enter the password
  3. For a new account, enter the following statement to grant the user select/query access to the DOCS table:
    grant select on docs to user_name;
  4. Grant/Revoke permissions as neccessary:

    To give the user the ability to upload documents, enter the following statements:

    grant insert on docs to user_name;

    grant select on docs_seq to user_name;

    To give the user the ability to edit document notes, enter the following statement:

    grant update on docs to user_name;

    To give the user the ability to delete documents, enter the following statement:

    grant delete on docs to user_name;

    To prevent the user from uploading documents, enter the following statements:

    revoke insert on docs from user_name;

    revoke select on docs_seq from user_name;

    To prevent the user from editing document notes, enter the following statement:

    revoke update on docs from user_name;

    To prevent the user from deleting documents, enter the following statement:

    revoke delete on docs from user_name;

4. Starting and Logging into PDFKeeper

Description
The PDFKeeper application is used to locate PDF document records in the database by Title, Author, Subject, Keywords, document notes, date and time added to the database, and any text within the PDF document.

The following tasks can be performed on any PDF document record that has been returned during a search:

  • View the document.
  • Save the document to disk.
  • Edit document notes.
  • View the document keywords.
  • Delete selected database records.
In addition, you can edit the Information Properties of PDF documents and upload them to the database.
How to start and login to PDFKeeper
PDFKeeper will function with User privileges and can be started by either:
  • Clicking the PDFKeeper desktop icon.

  • Clicking Start | All Programs | PDFKeeper | PDFKeeper

When prompted; enter the database Username, Password, and Data Source. The Data Source must be specified as follows: host:port
  • host is the host name or IP address of the computer that is running Oracle Database XE Server. If connecting locally to Oracle Database XE Server, the hostname or IP address must be localhost or 127.0.0.1
  • port (optional) is the TCP port number on which the Oracle Net listener is listening. If not specified, the default port number 1521 is assumed.

5. Document Searching

Description
PDF document records can be searched by Title, Author, Subject, Keywords, document notes, date and time added to the database, and any text within the PDF document.

The following tasks can be performed on any PDF document record that has been returned during a search:

  • View the document.
  • Save the document to disk.
  • Print document notes.
  • Edit document notes.
  • View document keywords.
  • Delete selected database records.
How to search for documents
From the main dialog; enter the Search Text or select a previously entered Search Text, and then click Search or press the Enter key.
Search Text examples:

about(insurance)

Will return documents aboutinsurance.

The about operator supports single words, phrases, and can be combined with other operators, such as AND, OR, or NOT. Any query operators included in a phrase will be ignored. However, you can use other operators, such as AND, OR, or NOT; to combine ABOUT queries with word queries.

about(insurance) and banking

Will return documents aboutinsurance, and banking.

about(insurance) and about(banking)

Will return documents about insurance, and about banking.

about(insurance) not about(homeowners)

Will return documents aboutinsurance, and not about homeowners.

auto and home

auto & home

Will return documents that contain auto and home.

maine=vermont is a new england state

maine equiv vermont is a new england state

Will return documents that contain either the phrase; maine is a new england state, or vermont is a new england state.

?product

Will return documents that match a fuzzy expansion of product.

insurance ~ auto

insurance not auto

Will return documents that contain insurance and not auto.

cars | trains

cars or trains

Will return documents that contain cars or trains.

!edgartown

Will return documents that sound like edgartown.

$produce

Will return documents that contain the same root word, as produce.

grocer%

Will return documents that contain the term, beginning with grocer.

_rain

Will return documents that contain the term, ending in rain.

Notes on special characters:
  • % is a multiple wildcard character.
  • _ is a single wildcard character.
  • Parenthesis characters () can be used to group terms and operators.
  • Braces {} can be used to escape a string of characters or symbols.
  • A backslash \ can be used to escape a single character.
  • Common characters, such as - (dash) and & that are part of a string, need to be escaped. For example, a string 2009-10-01 needs to be specified as 2009\10\01. With the BASIC_LEXER, a printjoin attribute is used requiring any - (dash) in a string to be escaped with an _ (underscore). For example, a string 2009-10-01 needs to be specified as 2009_10_01.
Sorting the results
When search results are returned, the sort order is by ID. Clicking on a list view column header will sort the results by the column selected, in ascending order; clicking on the same column header a second time will sort the results by the column selected, in descending order.
Refreshing the list view
To refresh the list view, using the last used sort order and search text; select the View menu, and then Refresh. Note, the F5 key will also refresh the list view.
Document Viewing
To view the contents of a document returned from a search, double click the document list view record. The document will be opened with the bundled, third-party PDF viewer. Note, the PDF document will remain cached on disk until PDFKeeper is closed out.
If your operating system supports the Encrypting File System (EFS), the PDF document will be encrypted. The following operating systems do not support EFS: Windows XP Home, Windows Vista Home Basic and Premium, and Windows 7 Starter, Home Basic and Premium.
Save a PDF document to disk without viewing
To save a PDF document that was returned from a search to disk, without having to view it:
  1. From the list view, select the PDF document record.
  2. Select the File menu, and then select Save PDF Document to Disk; or you can use the keyboard shortcut key Ctrl+S.
  3. Browse to the desired location.
  4. Modify the file name, if necessary.
  5. Select Save.
Printing Document Notes
To print document notes for any document record returned from a search:
  1. From the list view, select the PDF document record.
  2. Select the File menu, and then select Print Document Notes; or you can use the keyboard shortcut Ctrl+P.
  3. Select the Printer to use, and then select Print.
Editing Document Notes
Document Notes is where you can store information related to a selected document record. To edit document notes for any document record returned from a search:
  1. From the list view, select the PDF document record.
  2. Select the Document Notes tab.
  3. If you need to add a Date/Time stamp to the text box: select the Edit menu, and then select Insert Date/Time into Document Notes; or you can use the keyboard shortcut Ctrl+T.
  4. Make the necessary edits, and then select Update.
Viewing Document Keywords
To view the document keywords that were extracted from the PDF document InfoDictionary:
  1. From the list view, select the PDF document record.
  2. Select the Document Keywords tab.
Deleting documents
Any document returned from a search can be deleted from the database by:
  1. Clicking on the checkbox for each record that you wish to delete; to select all documents in the list: select the Edit menu, and then select Check all Documents; or you can use the keyboard shortcut Ctrl+C.
  2. Select the Edit menu, and then select Delete all Checked Documents; or you can use the keyboard shortcut Ctrl+D.
  3. When prompted, select Yes or press Enter to perform the delete.

6. Information Properties Editor

Description
Information Properties Editor is used to edit the Title, Author, Subject, and Keywords of one or more PDF documents prior to being uploaded to the database. The Author and Subject entry fields are linked to the database, to allow for selecting an existing Author and Subject.
Note, any PDF document that is protected by an OWNER or USER password cannot be edited. For more information, see the Troubleshooting section in the User FAQ.
How to edit the PDF Information Properties
To edit the PDF Information Properties of one or more PDF documents, perform the following:
  1. From the main form; select the Tools menu, and then select Information Properties Editor; or you can use the keyboard shortcut Ctrl+I.
  2. Browse to the desired location.
  3. Select one or more PDF files to be edited.
  4. Select Open.
For each PDF document selected:
  1. If necessary, view the PDF document by selecting View, which will open it with the bundled, third-party PDF viewer.
  2. Enter the document Title.
  3. Enter or select the Author of the document.
  4. Enter or select the Subject of the document.
  5. If necessary, enter document Keywords.
  6. If you want the PDF document displayed after saving, check the box labeled After saving, open PDF document in viewer.
  7. Select Save. Note, the original PDF document will be preserved in its original location.
  8. The updated PDF document is now ready to be uploaded into the database.
Information Properties Editor folder watcher
A folder watcher can be enabled to watch a selected folder, calling the Information Properties Editor for any PDF document that is created in this folder. When an event is detected, the absolute name of the PDF document is added to a queue and the Information Properties Editor will be called, unless it is still open from a previous event. The user will have the opportunity to edit each document that is in the queue. Note, it is highly recommended that a dedicated folder be used with the folder watcher.
To select the folder to be watched, perform the following:
  1. From the main form menu; select Tools, Folder Watcher, Information Properties Editor, and then Select Folder.
  2. Select the folder to be watched, and then select OK.
  3. To enable the folder watcher immediately, select Yes when prompted, otherwise select No.
To enable the folder watcher, perform the following:
  1. From the main form menu; select Tools, Folder Watcher, Information Properties Editor, and then Enable.
To disable the folder watcher, perform the following:
  1. From the main form menu; select Tools, Folder Watcher, Information Properties Editor, and then Disable.
NOTE: While the folder watcher is enabled, you will see a yellow folder object on the bottom right corner of the main form. Problems could arise when using this feature to watch a folder on a network drive or a very active folder, resulting in missed events. For troubleshooting, all exception errors are logged to %USERPROFILE%\Local Settings\Application Data\PDFKeeper\FolderWatcher.logon Windows XP or %LOCALAPPDATA%\PDFKeeper\FolderWatcher.log on Windows Vista and 7.

7. Document Uploading

Description
PDFKeeper will upload one or more, selected PDF documents, into the docs table. However, any PDF document that does not have a title, author, or subject; specified in the PDF Information Properties (InfoDictionary), will not be uploaded.
Note, any PDF document that is protected by an OWNER or USER password will not be uploaded. For more information, see the Troubleshooting section in the User FAQ.
How to upload documents
To upload one or more PDF documents, perform the following:
  1. From the main form; select the Tools menu, and then select Upload; or you can use the keyboard shortcut Ctrl+Shift+U.
  2. Browse to the desired location.
  3. Select one or more PDF files to be uploaded.
  4. Select Open.
Important Information
  • To prevent data loss, the upload is designed to not delete source PDF files.
